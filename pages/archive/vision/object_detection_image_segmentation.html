YOLO

YOLO →\rightarrow→ You Only Look Once

  • Simplest object detection architecture, not the most accurate though
  • Could be a good choice for real-time systems due to its fast prediction time.

YOLO →\rightarrow→ ConvNet backbone + object detection head & classification head

  • Create a N×MN \times MN×M grid.
  • Predicts x,y,w,h,c,classx, y, w, h, c, \text{class}x,y,w,h,c,class
  • x,yx, yx,y: position of bounding box center relative to the grid cell’s center
    • Activation function is tanhtanhtanh which predicts [−1,1][-1,1][−1,1]
  • w,hw, hw,h: % of image size
    • Activation function is sigmoid\text{sigmoid}sigmoid which predicts [0,1][0,1][0,1]
  • ccc: confidence score for a bounding box (whether an object is detected or not)
    • Activation function is sigmoid\text{sigmoid}sigmoid which predicts [0,1][0,1][0,1]
  • class\text{class}class: object class
    • Activation function is softmax\text{softmax}softmax

Loss function

  • The number of detection boxes per grid cell is a parameter.
  • In order to compare each detection box with ground truth, it uses the intersection over union (IoU) metric.

IoU=Area of intersectionArea of union \begin{aligned} \text{IoU} = \frac{\small{\text{Area of intersection}}}{\small{\text{Area of union}}} \end{aligned} IoU=Area of unionArea of intersection​​

\begin{aligned} \end{aligned}

  • Ground truth boxes are assigned to grid cells by their centers and to the prediction boxes within the grid cells by IoU\text{IoU}IoU.

Here are different components of the loss function. The final loss function is the addition of all these components.

  • Object presence loss: Each grid cell that has a ground truth box computes:
    Lobj=(1−C)2 \begin{aligned} \mathcal{L}_{obj} = {(1-C)}^2 \end{aligned} Lobj​=(1−C)2​
  • Object absence loss: Each grid cell that does not have a ground truth box computes:
    Lnoobj=(0−C)2=C2 \begin{aligned} \mathcal{L}_{noobj} = {(0-C)}^2 = C^2 \end{aligned} Lnoobj​=(0−C)2=C2​
  • Object classification loss: Each grid cell that has a ground truth box computes:
    Lclass=cross entropy(p,p^) \begin{aligned} \mathcal{L}_{class} = \text{cross entropy}(p, \hat{p}) \end{aligned} Lclass​=cross entropy(p,p^​)​
  • Bounding box loss: Each predicted box/ground truth box pairing contributes:
    Lbox=(x−x^)2+(y−y^)2+(w−w^)2+(h−h^)2 \begin{aligned} \mathcal{L}_{box} = {(x-\hat{x})}^2 + {(y-\hat{y})}^2 + {\small{(}\sqrt{w}-\sqrt{\hat{w}}\small{)}}^2 + {\small{(}\sqrt{h}-\sqrt{\hat{h}}\small{)}}^2 \end{aligned} Lbox​=(x−x^)2+(y−y^​)2+(w​−w^​)2+(h​−h^​)2​

Note: Difference in box sizes is computed on the square roots of the dimensions. This is to mitigate the effect of large boxes, which tend to overwhelm the loss.

Weighted loss

A common problem in object detection losses is that small losses from numerous cells with no object in them end up overpowering the loss from a lone cell that predicts a useful box. Weighting different parts of the loss function can alleviate this problem to some degree.

The empirical weights suggested by the authors of YOLO paper are:

λobj=1,λnoobj=0.5,λclass=1,λbox=1\lambda_{obj} = 1, \lambda_{noobj} = 0.5, \lambda_{\text{class}} = 1, \lambda_{\text{box}} = 1λobj​=1,λnoobj​=0.5,λclass​=1,λbox​=1

YOLO Limitations

  1. YOLO predicts a single class per grid cell. It will not work well if multiple objects of different kinds are present in the same cell.
  2. A fixed grid resolution imposes strong spatial constraints on what the model can do. YOLO models typically do not do well on collections of small objects, like a flock of birds, without careful tuning of the grid to the dataset.
